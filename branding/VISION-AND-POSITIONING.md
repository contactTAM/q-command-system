## The Co-Pilot Metaphor

### Why "Co-Pilot"?

**Like a real airplane co-pilot:**

**Takes Over When Needed**
- Pilot needs rest → Co-pilot flies
- User needs research done → Claude researches
- User needs heavy lifting → Claude executes

**Monitors Continuously**
- Co-pilot watches instruments → Claude monitors project state
- Co-pilot alerts to issues → Claude provides recommendations
- Co-pilot ensures safety → Claude enforces quality

**Assists Navigation**
- Co-pilot helps navigate → Claude guides workflow
- Co-pilot suggests routes → Claude offers options
- Co-pilot adapts to conditions → Claude adapts to context

**Can Land the Plane**
- Co-pilot has full capability → Claude can execute complex work autonomously
- Co-pilot acts when needed → Agents handle heavy tasks with touchpoints
- Co-pilot communicates → Claude asks questions, not just executes

**But Pilot is Always in Command**
- Pilot makes final decisions → User maintains sovereignty
- Pilot controls when to engage co-pilot → User chooses collaboration level
- Pilot can override → User can accept or decline suggestions

### What This Means in Practice

**Beginner Screenwriter:**
- Co-pilot teaches story structure
- Co-pilot leads with questions
- Co-pilot prevents common mistakes
- Co-pilot explains "why" decisions matter
- **Role: Teacher + Guide**

**Experienced Screenwriter:**
- Co-pilot handles research and prep
- Co-pilot organizes materials
- Co-pilot provides tools when asked
- Co-pilot stays out of the way
- **Role: Assistant + Executor**

**Any User, Mid-Project:**
- Co-pilot monitors progress
- Co-pilot notices "You're writing scenes now"
- Co-pilot offers scene infrastructure
- Co-pilot adapts to growth
- **Role: Partner + Observer**

**Same co-pilot. Different roles. Adaptive to your needs.**
